groff; part one

This is an introduction to the groff typesetting system; derived from the original troff 1 and roff 2, groff is the UNIX version (of roff) that is default installed on most linux systems. Similar to LaTeX, it doesn't require a graphical front-end and it converts text to a document file (usually pdf). While LaTeX is inarguably more robust than groff, there are some clear advantages to groff including simplicity and speed of use. In contrast to LaTeX, it is a lot more minimal, with compile speeds being a lot faster with the same document. Using the -ms macros, I have conducted a test comparing the speed of the compile times. As you can see here, the average time for the groff document to compile is nearly half the average time it takes for the LaTeX document to produce the exact (content) same document. In this test I made sure to include mathematics to demonstrate the similarity that (basic) LaTeX and groff have. While groff is a lot faster than LaTeX, it is almost too minimal to a point that it is hard to start using. While LaTeX uses packages such as amsmath or mhchem, groff uses preprocessors which are applied with the compilation command, some include eqn or chem (respective to the LaTeX packages). Groff is macro-based and uses different macro packages for different uses. I personally use -ms macros simply because it is the first I used. Other popular macro packages include -mom and -man.
